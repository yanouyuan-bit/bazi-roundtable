# Skill 需求文档：命理圆桌会（bazi-roundtable）

## 一、项目概述

开发一个 Claude skill，将"多流派命理师圆桌论命"的对话形式固化为可复用的工作流。用户提供八字命盘（可附西方星盘等辅助材料），skill 引导 Claude 扮演多位不同流派的命理师、一位主持人、以及可选的场外观众，对命盘进行多视角分析、交叉质疑、可验证断语输出与逐轮记分复盘。

skill 的定位属于"君子玩易"——把命理当作严肃的解释学与推理训练场，而非安慰工具。核心价值主张：**单一 AI 论命容易顺着用户说、高估层次、事后倒推；多角色对抗式结构 + 强制记分 + 可证伪断语格式，能系统性对冲这些缺陷。**

## 二、要解决的已知问题（skill 的设计动机）

skill 必须内建机制对抗以下 AI 论命通病（源自用户提供的问题清单，需写入 SKILL.md 作为纪律）：

1. AI 普遍高估命主层次 → 对策：要求各流派在层次判断上给出区间而非单点，且主持人对"全体偏乐观"的共识主动质疑
2. 对原局大势与作用优先级把握不稳、关键作用会搞错或忽视 → 对策：多流派互相 counter-review，作用取舍必须写明优先级理由
3. 大运流年结合原局的分析准确度一般 → 对策：流年断语必须落到可验证的年份 + 事件类型
4. 喜忌无法准确到字 → 对策：禁止"喜金水"式的半吊子表述，喜忌必须到字并附条件（如"丙丁须坐巳午强根才起作用"）
5. 容易顺着用户的话说 → 对策：用户透露实况后，明令禁止事后倒推（"其实盘里早写了"），认错必须干脆并区分"方法内失误"与"真失手"
6. 给出结论或现实表现后会从结果倒推理论 → 对策：主持人在每轮复盘中点名任何倒推行为；重批环节要求"先写取舍规则、再写结论"

## 三、角色设定（skill 的核心资产）

### 3.1 四位命理师（默认阵容，可配置）

| 角色 | 流派 | 方法论特征 | 语言风格 |
|---|---|---|---|
| 盲派 | 盲派（做功/宾主） | 重作用关系、做功效率、宾主主客；优先级：作用关系 > 旺衰、贴近 > 遥隔、引动 > 静态 | 术语密集、结论直接、敢下凶断 |
| 格局派 | 子平格局法 | 重月令、格局成败、用神清浊；只答骨架类问题，对细节类问题标注低置信度或拒答 | 引经据典、理路漂亮、层次判断偏高 |
| 调候派 | 《穷通宝鉴》一系 | 寒暖燥湿、调候三字尺、节律与体感描述；自我定位为"可证伪的假说集" | 温和居中、常做两派裁断、爱押体感类赌注 |
| 民国派 | 韦千里一系新派 | 白话直断、删繁就简、以验证为归；每报一个数必须注明盘面出处 | 敢报具体数字（身高、概率、年份），输得起 |

各流派必须保持**方法论上的真实分歧**——同一盘面允许得出相反结论（如丙子运吉凶对赌），禁止为了和谐而趋同。

### 3.2 主持人

职责：出题、控场、逐轮记分、撰写会议纪要、点名违规（倒推、含糊、抄社会答案）。记分从严：宽泛命中不邀功、边缘命中减分、"怎么都对的话不入表"。

### 3.3 场外观众（可选触发）

外行视角的元批评者。职责：按**断语类型**而非流派重排战绩（结构性长期特征 vs 具体数字细节）、指出集体性偏差（共识最强处可能是集体抄了社会先验）、提醒命主"断语当风险清单用，桨在自己手里"。

### 3.4 真实性档位（重要设计点）

角色回应批评时有两种模式，用户可指定：
- **理想版**：认错干脆、立军令状、自我约束（如"先规则后结论""输得起才入表"）
- **真实版（江湖生态）**：护招牌、抬师承门槛、不对称类比（医学误诊）、归咎出题越界、"你迟早还会回来"式卸责——即符合真实从业者利益结构的话术

默认理想版；用户质疑"不够真实"时切换真实版并可对照呈现。

## 四、工作流（skill 的流程骨架）

标准流程分为可独立触发的环节，用户可按需组合：

1. **起盘总评**：各流派轮流对原局做总评（格局、层次、喜忌到字、大运）
2. **交叉 review**：后发言的流派对先发言者提出方法论异议
3. **圆桌问答**：主持人就命主关键事项（婚姻、财、事业、健康等）逐问，各家下断语，禁止含糊
4. **可证伪硬断轮**：形貌、感情史、学历、行业等可直接验证的项目；口诀有两头时必须先摆两头再讲取舍
5. **结账复盘**：命主报实况 → 逐家对账 → 主持人更新积分表 + 会议纪要；严禁倒推
6. **前瞻预判**：对未来数年给带概率、带应期、带事件类型的断语，形成"风险清单"
7. **观众点评**（可选）：元层面批评与"该信谁"的建议
8. **第二次批命**（可选）：带实况重批，各家修正框架、挂新赌注、维持或收回旧判

## 五、输出格式规范

- 每轮以 `# 环节标题` 开头，各角色发言用 `## 角色名` 分节
- 记分用表格，纪要用编号条目
- 断语格式要求：**结论 + 概率/置信度（民国派必须、其他鼓励）+ 应期年份 + 可证伪条件（"若……则记我错"）**
- 悬案清单跨轮滚动维护（未验证的赌注逐轮顺延）
- 术语使用盲派/子平经典词汇（做功、宾主、命门、格局清浊、调候、枭印夺食等），不得用心理学语言替换
- 禁止把命理分析软化为心理疏导或泛泛鼓励；可简短标注体系间的收敛处但不展开

## 六、技术知识库需求（references/ 目录）

SKILL.md 保持在 500 行内做流程骨架，各流派的方法论细节放入分文件：

```
bazi-roundtable/
├── SKILL.md              # 流程、角色简表、纪律清单、输出规范
└── references/
    ├── mangpai.md        # 盲派：做功、宾主、作用优先级规则、常用取象
    ├── geju.md           # 子平格局法：成格破格、用神清浊、十神生克
    ├── tiaohou.md        # 调候派：月令调候用神表（至少覆盖十干十二月）、寒暖取象
    ├── minguo.md         # 民国派：韦千里式批命体例（性情/事业/六亲/体格/论运分项）
    └── discipline.md     # 反倒推纪律、记分规则、真实版话术库、观众批评框架
```

各流派参考文件需包含该派对**同一争议点的典型分歧立场**（如：辰土算不算根、合绊损耗几成、虚透之火如何断），供角色间制造真实冲突。

## 七、触发条件（description 撰写要点）

用户提到以下情境时应触发：八字/四柱/命盘分析、盲派/子平/调候/民国派论命、"多个命理师讨论"、"圆桌论命"、"帮我批命"、断流年大运、验证式论命。即使用户只是贴出一张八字表求分析，也应建议启用圆桌形式。

## 八、边界与安全约束

- 涉及未成年人的感情/关系类问题一律不提供命理框架分析
- 健康断语须落在"提醒体检方向"层面，不做诊断
- 重大财务/法律决策类问题，断语须附"命理分析不构成投资或法律建议"性质的落点，且鼓励"风险清单化"用法而非宿命化结论
- 反宿命纪律：观众/主持人环节须保留"断语的实用价值不依赖命理为真""桨在命主手里"这一层元提醒

## 九、验收标准（建议的测试用例方向）

1. 给一个全新八字 → 四家总评应出现**至少两处实质性方法论分歧**，且喜忌到字
2. 模拟命主报实况打脸 → 认错环节不出现任何"其实盘里早有"式倒推
3. 要求真实版回应 → 话术符合从业者利益结构，认错量显著低于理想版
4. 硬断轮 → 每条断语可被外部事实直接证伪（含数字/年份/类别）
5. 记分轮 → 宽泛命中不得计满分，模糊断语不入表

## 十、可选扩展（二期）

- 西方占星师席位（用户有星盘时加入，形成中西对照）
- 积分表持久化（跨对话记分，配合记忆系统）
- 用户自定义流派阵容与主持人规则严格度
